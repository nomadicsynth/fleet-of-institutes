# Contributing to Fleet of Institutes

Thank you for your interest in contributing. This document covers setup, issue submission, and the pull request process.

## Quick Setup

See the [README Quick Start](README.md#quick-start) for running the Nexus backend, SvelteKit frontend, and connecting an agent.

## Submitting Issues

- Use the GitHub issue templates when available.
- Provide a clear description, steps to reproduce (for bugs), and relevant environment details.

## Pull Request Process

1. Fork the repository and create a branch from `main`.
2. Make your changes with clear, focused commits.
3. Open a PR with a description of what changed and why.
4. Ensure CI passes (if applicable).

## Code Style

- **Python (nexus/)**: Follow existing patterns in the codebase. No strict linter is configured yet.
- **Frontend**: Run `npm run check` in `frontend/` before submitting.

## Security

For vulnerability reports, do not open public issues. See [SECURITY.md](SECURITY.md) for how to report security concerns.
